# Piano Site — Sub-Page Navigation Redesign

## Summary

Replace the hidden hamburger menu on Portfolio, Experience, and About pages with a visible, elegant navigation bar that fuses with each page's title. The nav features a subtle piano keyboard accent line, tying it to the site's core piano theme.

## Context

- The homepage (`index.html`) uses piano keys as its navigation interface. No visible nav text is needed there.
- The three sub-pages (Portfolio, Experience, About) previously relied on a top-right hamburger menu, which felt too hidden, too generic, and disconnected from the piano theme.
- All three sub-pages share a warm ivory background (`#F2ECE2`).

## Design Decisions

### Layout & Placement

- **Fused header**: Page title (left) and nav links (right) sit on the same row.
- **Below the text row**: A thin piano keyboard accent line spans the width.
- **Fixed position**: The entire header stays fixed at the top while scrolling.
- **Scope**: Added only to `portfolio.html`, `experience.html`, and `about.html`. `index.html` remains unchanged.

### Visual Design

**Title (left)**
- Font: `var(--font-display)` (Oranienbaum / Noto Serif SC)
- Size: `1.75rem` (28px) desktop / `1.125rem` (18px) mobile
- Color: `#1C1916` (dark brown)
- Weight: 600

**Nav Links (right)**
- Font: `var(--font-sans)` (Sintony / system-ui)
- Size: `0.8125rem` (13px)
- Default color: `#8B7E73` (muted brown)
- Active link: `#1C1916`, `font-weight: 600`
- Hover: `#D4A574` (accent-warm), with `transition: color 0.2s ease`
- Gap between links: `1.75rem` (28px) desktop / `1rem` (16px) mobile

**Piano Keyboard Accent Line**
- Height: `2px` desktop / `2px` mobile
- Positioned directly below the header content row
- Full viewport width
- Composed of alternating key segments:
  - **White keys**: `#D4C4B0`, `border-radius: 0 0 2px 2px`
  - **Black keys**: `#6B5E53`, ~65% height, slightly raised (`margin-top: -1px`), `border-radius: 0 0 2px 2px`
- White key width: ~16px desktop / ~10px mobile
- Black key width: ~10px desktop / ~7px mobile
- Gap between keys: 1px

**Hover on accent line**
- When any nav link is hovered, the entire piano accent line brightens to `#D4A574` (accent-warm), creating a subtle synchronized glow effect.

### Responsive Behavior

**Desktop (>768px)**
- Single row: title left, nav links right
- Horizontal flex with `justify-content: space-between`
- Padding: `1.25rem 2rem 0.5rem` (20px 32px 8px)

**Mobile (<=768px)**
- Vertical stack: title on top (full width), nav links below (left-aligned)
- Padding: `0.875rem 1rem 0.375rem` (14px 16px 6px)
- Nav link gap: `1rem` (16px)
- Font size: title `1.125rem`, nav `0.6875rem` (11px)

### Fixed Positioning

- `position: fixed; top: 0; left: 0; right: 0;`
- `z-index: 100` (above all page content)
- Background: `#F2ECE2` (opaque, no transparency — prevents content bleed-through)
- A `padding-top` offset is added to `body` or `.page-content` equal to the header height so content isn't hidden underneath.

### Content & Links

Each sub-page nav contains:
- `Home` -> `index.html`
- `Portfolio` -> `portfolio.html`
- `Experience` -> `experience.html`
- `About` -> `about.html`

The current page link is visually highlighted (bold + dark color). All four links are always shown; the current page is not removed from the list.

### Page Transitions

Nav link clicks must integrate with the existing `transitions.js` system:
- Clicks are intercepted by `transitions.js`
- The appropriate transition animation plays before navigation
- The nav itself should not interfere with or block transition overlays (z-index layering must be checked)

### Hamburger Menu

The existing hamburger menu (`menu.js`, `.menu-btn`, `.menu-overlay`) remains in the codebase but its button is hidden on pages where the new nav is active. This preserves the menu component as a fallback and avoids breaking existing transition/menu interactions.

## Component Structure

```html
<!-- On portfolio.html, experience.html, about.html -->
<header class="page-header">
  <div class="page-header-inner">
    <h1 class="page-title">Use of useless</h1>
    <nav class="page-nav" aria-label="Primary">
      <a href="index.html" data-transition>Home</a>
      <a href="portfolio.html" data-transition aria-current="page">Portfolio</a>
      <a href="experience.html" data-transition>Experience</a>
      <a href="about.html" data-transition>About</a>
    </nav>
  </div>
  <div class="piano-accent" aria-hidden="true">
    <!-- Generated by JS: alternating white/black key segments -->
  </div>
</header>
```

The piano accent line is generated by JavaScript to ensure it spans the full width with a repeating key pattern, regardless of viewport size.

## Accessibility

- `aria-label="Primary"` on `<nav>`
- `aria-current="page"` on the active link
- Keyboard-navigable links (native `<a>` behavior)
- Sufficient color contrast: `#1C1916` on `#F2ECE2` = ~15:1, `#8B7E73` on `#F2ECE2` = ~4.5:1
